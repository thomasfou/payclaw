## Clawdbot/OpenClaw systemd user session fix

**Problem:** On some Ubuntu machines (especially via SSH), systemctl --user fails with:
```
Error: systemctl --user unavailable: Failed to connect to bus: No medium found
```

This breaks `clawdbot gateway start/restart/install` and `openclaw gateway start/restart/install`.

**Solution:**
```bash
export XDG_RUNTIME_DIR="/run/user/$(id -u)"
```

Then the normal gateway commands work:
```bash
systemctl --user status
openclaw gateway stop
openclaw gateway start
```

**Permanent fix â€” add to ~/.bashrc:**
```bash
echo 'export XDG_RUNTIME_DIR="/run/user/$(id -u)"' >> ~/.bashrc
```

**Alternative if systemd still won't work:**
```bash
sudo loginctl enable-linger $USER
# Then fully logout/login
```

**Workaround without systemd:**
```bash
tmux new -d -s clawdbot 'clawdbot gateway run'
```
